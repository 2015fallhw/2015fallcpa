GitHub使用

要使用GitHub必需先建立帳號，建立帳號後，就能夠有效的管理倉儲及上傳資料。

每次上傳資料，GitHub會記錄每次推送的版本，如果檔案出現問題，可以利用回朔，找回還沒出問題的檔案。

為什麼使用GitHub?

出社會後，如果總是把資料或要給客戶的設計圖放在隨身硬碟或電腦上，如過電腦和隨身硬碟出了問題，總不能更客戶說電腦壞了或資料檔案錯誤等跟客戶交差，這樣會影響客戶的心情或公司對你的信任，所以身為工程師，應該要將資料放入遠端，這樣會自己減少很多不必要的麻煩。

GitHub可以有有效的管理檔案，建立自己的倉儲，管理自己的資料。也可以利用協同，讓大家都可以管理一份資料。即使資料出問題了，也能有效的回朔到沒問題的資料，當客戶拿到的檔案出問題了，也能第一時間修改錯誤，讓客戶能交差。

利用leo編輯網頁資料，上傳到GitHub倉儲裡，再利用GitHub倉儲網址連結到個人的網頁。GitHub能廣泛運用在太多管理資料上，所以使用GitHub管理自己的資料能有效率的解決許多問題

Git基本操作

  • git clone 加入倉儲網址=複製倉儲資料
  • git checkout 加入分支名=切換分支
  • git add -A=拍快照到資料庫
  • git commit -m=提交
  • git push = 推送
  • git config --global user.email 帳號
  • git config --global user.name 使用者
  • git pull = 匯入資料

Git基本操作

  • git branch 分支名=建立分支
  • git branch=查看分支
  • git reset --hard = 回朔
